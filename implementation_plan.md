# Tech Stack & Architecture Plan: Sanjivani Farms

This document outlines the finalized technical stack and the implementation phasing for the Sanjivani Farms booking and management system. The goal is to build a secure, highly scalable, and 100% free (or mostly free) system that supports dynamic pricing, real-time dashboards, and future integrations.

## 1. Core Technology Stack

Here are the precise technologies we will use for the system:

*   **Frontend Framework:** **Angular (Latest Version)**
    *   *Why:* Highly structured, fast, and scalable. Excellent for building large-scale dashboards and complex dynamic pricing forms.
*   **Styling & Animations:** **Tailwind CSS + GSAP (GreenSock)**
    *   *Why:* Tailwind allows for rapid, responsive UI development. GSAP provides premium, luxurious micro-animations to ensure the resort website feels high-end.
*   **Database:** **Firebase Firestore (NoSQL)**
    *   *Why:* Real-time data sync to the owner dashboard. Generous free tier (50,000 reads/day) that easily handles 100-500 enquiries daily.
    *   *Strategy:* Implement data pagination (loading 50 enquiries at a time) to strictly protect the free tier limits.
*   **Authentication:** **Firebase Authentication**
    *   *Why:* Secure, easy integration with Firestore. Allows secure login for the Owner/Admins to manage the dashboard.
*   **Media Storage:** **Cloudinary**
    *   *Why:* Offload heavy media (photos, videos) from the database. Cloudinary automatically optimizes images for fast loading.
    *   *Strategy:* Only store the `URL` strings generated by Cloudinary inside Firestore to keep database size tiny.
*   **Backend Automation & APIs:** **Firebase Cloud Functions (Node.js)**
    *   *Why:* Needed when we eventually integrate the WhatsApp Business API. It securely listens for new entries in Firestore and triggers custom logic.
*   **Hosting:** **Vercel or Firebase Hosting**
    *   *Why:* Both provide excellent free tiers, global CDN caching for fast load speeds, and seamless deployment for Angular.

## 2. System Architecture & Data Flow

How the pieces fit together:

1.  **Guest Experience (Website View):**
    *   Guest visits the Angular site.
    *   Images/Reels load lightning-fast from **Cloudinary**.
    *   Guest selects details and submits an enquiry.
2.  **Data Capture & Storage:**
    *   Enquiry data (Name, Phone, Dates, Cottage Type, Total Price) is sent to **Firestore**.
3.  **Owner Management (Admin Dashboard View):**
    *   Owner logs in using **Firebase Auth**.
    *   Angular reads the latest data from **Firestore** in real-time. The new enquiry instantly pops up on the dashboard.
    *   Owner can approve/reject, update pricing, or edit Cloudinary image links directly from this UI.
4.  **Future Integration (WhatsApp):**
    *   When the Owner clicks "Confirm" on the dashboard, a **Firebase Cloud Function** fires to talk to the WhatsApp Business API (e.g., AiSensy) and sends the booking confirmation to the guest.

## 3. Development Phases

### Phase 1: Foundation & Data Modeling
*   Set up the Angular workspace.
*   Configure Firebase (Auth & Firestore) projects.
*   Set up Cloudinary account and link upload logic.
*   Define the Firestore Database Schema (Structure for `Enquiries`, `Cottages`, `Settings`).

### Phase 2: Core User Interface (The Website)
*   Build the premium landing page, immersive cottage views, and high-quality image galleries using GSAP and Tailwind.
*   Implement the Dynamic Pricing Engine (Angular Signals estimating real-time calculated costs based on guest counts and cottage types).
*   Build the Booking/Enquiry Submission form that writes to Firestore.

### Phase 3: The Owner Dashboard
*   Create a secure login portal using Firebase Auth.
*   Build a real-time data table to list all Enquiries (with pagination to save reads).
*   Build the Content Management tools (UI to update Prices, Capacities, and upload/manage Cloudinary images).

### Phase 4: Automation & WhatsApp (Future Scope)
*   Integrate Firebase Cloud Functions to connect the final WhatsApp automated replies when the owner is ready.
